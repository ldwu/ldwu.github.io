使用七牛js SDK遇到的坑

发表于 2016-07-13   |   分类于 JavaScript   |     |   阅读次数

背景

项目中需要用到云存储服务，公司决定使用七牛的服务，七牛提供了服务端的API也提供像JS SDK可供web前端调用的方式，考虑到服务器的性能及为了降低服务器负载，上传文件选择从后端授权，前端直接上传文件的方式，上传成功后再通知后端做相应处理。

具体实现

由于业务的需求，我们需要在一个页面多次调用upload_init，去绑定多个上传的按钮，我们就会发现除非我们使用一个全局变量key才能成功，意思是第二次调用七牛的Qiniu.uploader时，会把第一次的key给污染了，导致UploadComplete，Key的中key不是同一个对象。即使采用闭包的方式也没有用。期待js高手答疑~~~~我懵！
最有可能的是七牛代码对key做了赋值处理，而且被赋予的是一个具有全局属性的变量。
